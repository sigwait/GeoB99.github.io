<h2 id="native-api-user-mode-layer-ntdll-layout-source-tree">Native API user-mode layer (NTDLL) layout source tree</h2> <p>The Native API, serving the purpose as a system call interface for subsystem environments, provides the Rtl section known as the Run-Time library as well as Loader and SxS (Side-by-Side Assembly) module source codes. The base library service of the Windows subsystem, <strong>Kernel32.dll</strong>, makes a call into NTDLL which in turn invokes a specific kernel mode service in the NT kernel to accomplish a request. <strong>User32.dll</strong> and <strong>Gdi32.dll</strong> libraries are exceptions as these make calls which are then trapped into kernel mode in Win32k.sys, not NTDLL!</p> <p>With that said, whenever an application makes a function call to <code>IsProcessInJob</code> in <strong>Kernel32.dll</strong> library the call flow operation goes through NTDLL and then into the kernel with the path leading to <code>NtIsProcessInJob</code> call.
